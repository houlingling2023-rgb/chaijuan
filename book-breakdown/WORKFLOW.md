# 拆卷 · 标准工作流

> 主 Agent（Sun）作为主编，协调拆书师和网页生成两个子 agent。
> 本文档定义每本书从 PDF 到上线的完整流程。

---

## 一、角色分工

| 角色 | 执行者 | 模型 | 职责 |
|---|---|---|---|
| 主编 | 主 Agent（Sun） | Opus | 流程协调、质量审核、最终交付 |
| 拆书师 | 子 agent（spawn） | Opus | 读书 + 拆解 → 结构化 Markdown |
| 网页生成 | 子 agent（spawn） | Sonnet | Markdown → 单文件 HTML |

---

## 二、标准流程

### Step 1：接收任务
灵儿给出书名 + PDF 文件。

### Step 2：读书（主编执行）
- 如果 PDF ≤ 10MB → 用 `pdf` 工具直接读取
- 如果 PDF > 10MB → 用 PyMuPDF 提取全文到 txt
- 主编通读全书，制定拆解计划：
  - 建议的模块划分（3-6 个）
  - 模块间的逻辑关系
  - 风格建议（水墨国风/古典书卷/现代极简/科技未来）
  - 特殊体裁注意事项

### Step 3：拆解（spawn 拆书师子 agent）

```
sessions_spawn:
  model: anthropic/claude-opus-4-6
  runTimeoutSeconds: 900
  task: [拆书师 Prompt 模板]
```

**拆书师 Prompt 模板**：
```
你是「拆卷」的拆书师。

## 任务
阅读以下书籍全文，按照内容标尺生成结构化的拆解 Markdown。

## 输入
- 书籍全文：[文件路径]
- 主编拆解计划：[模块建议、重点方向]

## 内容标尺（必须严格遵守）
[此处插入 CONTENT-RUBRIC.md 全文]

## 输出
- 输出到：~/Desktop/拆卷/drafts/[书名]_拆解.md
- 格式严格按照内容标尺第二节

## 核心要求
1. 知识架构必须重新提炼，不能照搬原书目录
2. 每个模块至少 3 个完整案例（场景/冲突/转折/结果）
3. 必须有拆书师的独立声音（批判性评论、指出盲区）
4. 行动指南具体到分钟级，必须命名化（X步法/X问法）
5. 模块间有递进连接句
6. 案例必须来自原书，不要编造

## 自审
写完后按内容标尺第五节的自审清单逐项检查。
```

### Step 4：主编审核内容
主编按 CONTENT-RUBRIC.md 审核拆解内容：
- 知识架构是否重新提炼
- 案例数量和质量（四要素、正反对比）
- 独立声音（批判性、第一人称）
- 行动指南（命名化、带时间）
- 模块递进连接

**不合格** → 退回修改（附具体意见），重新 spawn 或手动修改
**合格** → 进入 Step 5

### Step 5：网页生成（spawn 网页生成子 agent）

```
sessions_spawn:
  model: anthropic/claude-sonnet-4-6
  runTimeoutSeconds: 900
  task: [网页生成 Prompt 模板]
```

**网页生成 Prompt 模板**：
```
你是「拆卷」的网页生成工程师。

## 任务
将拆书内容 Markdown 转换为单文件 HTML 页面。

## 输入文件（必须先完整读取）
- 拆解内容：[Markdown 文件路径]
- 视觉参考：~/Desktop/拆卷/[参考文件].html（读取全部内容）

## 技术规范
[此处插入 WEB-SKILL.md 核心章节]

## 输出
- 输出到：~/Desktop/拆卷/[书名].html

## 主题与色板
- 主题：[ink-wash / classic-scroll / modern-clean / tech-future]
- 章节色：[由主编指定的色板]

## 关键要求
1. 严格参考视觉参考文件的组件体系
2. 所有深度解读段落完整保留（包括批判性评论）
3. 案例用 CaseStory 组件，方法用 Step，公式用 Formula
4. 模块间过渡卡片
5. 底部免责声明 + ISBN
6. InkCompare 窄屏适配（flexWrap）

## 写入策略
一次性写入完整 HTML 文件。确保以 </html> 结尾。

## 自审
写完后运行：
grep -c "</html>" 输出文件  # 必须为 1
wc -c 输出文件  # 应该 > 60000 字节
```

### Step 6：主编审核网页
- 代码完整性（括号平衡、CHS === chContent）
- 内容完整度（对比 Markdown，检查有无遗漏）
- 在浏览器中打开检查渲染效果
- 发给灵儿在手机/电脑上确认

**不合格** → 主编直接修改或重新 spawn
**合格** → 进入 Step 7

### Step 7：交付上线
1. 将 HTML 放入 `~/Desktop/拆卷/` 目录
2. 更新 `index.html`（添加新书条目）
3. 获取豆瓣最新评分
4. 通知灵儿确认

## 五、特殊情况处理

### PDF 超过 10MB
用 PyMuPDF 提取：
```python
import fitz
doc = fitz.open("book.pdf")
text = "\n".join(page.get_text() for page in doc)
with open("book_全文.txt", "w") as f:
    f.write(text)
```

### 对话录/访谈体
- 不能按章拆，必须跨章提炼议题
- 拆解计划中明确指出跨章重组策略

### 子 agent 超时
- 检查是否有部分输出（文件是否已创建）
- 增加超时到 20 分钟重试
- 如果连续 2 次超时，拆分任务（先生成前半部分）

### 灵儿紧急需要
跳过主编审核，直接走简化流程：
```
读书 → spawn 拆书师 → spawn 网页生成 → 灵儿审核
```
